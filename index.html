HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Metadata works behind the scenes and contains important information such as the character set, description, keywords, author, and viewport.
  • "Body" contains all of the smaller elements that will be displayed on the web app.
  • "!DOCTYPE" this declaration tells the browser what document type it should expect when it is opened
  • Hypertext Markup Language (HTML) is used to create the structure of a webpage like text, image, header, and list elements.

CSS

  • Enter your CSS notes here
  • Inline CSS applied to a specific HTML tag and style a particular page element only. Used for quick and permanent changes but not flexiable.
  • Internal style sheet used in the head section of the HTML. Will apply only to that certain page.
  • External CSS style sheet is preferred because it allows developers to keep all their CSS rules in a separate file to make design changes easier.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property
  • Example: Color: Blue; ....CSS property: Color ....Value Property: Blue

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Git is a version control system that lets you manage and keep track of each version of your projec
  • Repository contain all of the files and revision histories for each project hosted on GitHub.
  • Issues used to track bugs, but they can also be used to collect user feedback and organize tasks.
  • Pull/push merging changes made to the repository files into the local copy you’re working on is called “pulling.” A “push” is the reverse, updating the repository files with changes you’ve made to your local copy.
